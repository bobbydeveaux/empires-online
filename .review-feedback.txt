### CRITICAL: `broadcast_event()` will silently fail in production

**File:** `backend/app/services/game_broadcast.py:123-132`

```python
def broadcast_event(game_id: int, message: Dict[str, Any]) -> None:
    loop = asyncio.get_event_loop()
    loop.create_task(_broadcast(game_id, message))
```

This function is synchronous and is passed to `background_tasks.add_task(broadcast_event, ...)`. FastAPI runs sync background task callables in a **threadpool**. From that worker thread, `asyncio.get_event_loop()` will **not** return the main running event loop — it will either raise a `DeprecationWarning` and return a new (non-running) loop, or raise `RuntimeError` in Python 3.12+. Either way, `create_task()` schedules work on a loop that is never run, so **broadcasts silently never execute**.

**Fix:** The simplest solution is to pass the async `_broadcast` function directly to `background_tasks.add_task()`, since FastAPI natively supports async background tasks:

```python
# In games.py, replace all occurrences of:
background_tasks.add_task(broadcast_event, game_id, message)

# With:
background_tasks.add_task(_broadcast, game_id, message)
```

Or alternatively, make `broadcast_event` async:

```python
async def broadcast_event(game_id: int, message: Dict[str, Any]) -> None:
    await _broadcast(game_id, message)
```

The unit test `test_broadcast_event_creates_task` passes only because the test already runs inside an async event loop (via `pytest.mark.asyncio`), which masks this issue.
